

# Domain-Driven Design (DDD)

### Учебный проект для оттачивания технической экспертизы и углубленного освоения архитектурных принципов через практическую реализацию DDD на PHP.

## Общая идея проекта

Общая идея модуля заказа (Order) в контексте DDD — это создание ограниченного контекста (Bounded Context), отвечающего за управление заказами, включающего бизнес-логику, связанные агрегаты, события и репозитории. Такой подход позволяет изолировать логику заказов, снижая сложность системы и обеспечивая четкое разделение ответственности. В рамках этого контекста заказ становится самостоятельной единицей, взаимодействующей с другими частями системы через определенные интерфейсы, что способствует созданию более гибкой и масштабируемой архитектуры.

### Цель модуля

Модуль заказа предназначен для управления процессом оформления, обработки и хранения заказов в системе. Он реализует бизнес-логику, связанную с заказами, отделяя её от инфраструктурных деталей и внешних интерфейсов.

### Основные принципы

- **Доменное моделирование**: В центре — сущность `Order` и связанные с ней объекты-значения (например, `OrderStatus`).
- **Ясная граница ответственности**: бизнес-логика реализована внутри доменного слоя (`Domain`), а инфраструктура — отдельно.
- **Команды и обработчики**: операции выполняются через команды (`CreateOrder`) и их обработчики, обеспечивая четкую последовательность действий.
- **Инверсия зависимостей**: инфраструктурные реализации внедряются через интерфейсы, что позволяет легко заменять базы данных, очереди и т.д.

### Пример структуры проекта, основные классы и компоненты:

```plaintext
src/                                # Корневая папка исходных кодов проекта
└── Contexts/                       # Контексты бизнес-логики (домены)
    └── Order/                      # Модуль "Заказы" (Order) - управление заказами
        ├── Domain/                 # Доменная логика (сущности, ценности, события, репозитории)
        │   ├── Entities/           # Сущности домена, основные бизнес-объекты (например, Order)
        │   │   └── Order.php       # Класс Order — основная сущность заказа
        │   ├── ValueObjects/       # Объекты-значения, используемые внутри сущностей (например, статус)
        │   │   └── OrderStatus.php # Статус заказа как объект-значение
        │   ├── Events/             # Доменные события, связанные с заказами
        │   │   └── OrderCreated.php # Событие "Заказ создан"
        │   └── Repositories/       # Интерфейсы репозиториев для работы с данными
        │       └── OrderRepositoryInterface.php # Интерфейс репозитория заказов
        ├── Application/            # Логика приложения, бизнес-операции, команды
        │   ├── Commands/           # Команды, инициирующие действия
        │   │   └── CreateOrder.php # Команда "Создать заказ"
        │   ├── Handlers/           # Обработчики команд
        │   │   └── CreateOrderHandler.php # Обработчик команды создания заказа
        │   ├── Services/           # Сервисы бизнес-логики, использующие репозитории
        │   │   └── OrderService.php # Логика работы с заказами
        │   └── DTOs/               # Data Transfer Objects — объекты передачи данных
        │       └── OrderDTO.php    # DTO заказа
        ├── Infrastructure/         # Инфраструктурные компоненты, интеграции
        │   ├── Persistence/        # Реализации репозиториев (например, база данных)
        │   │   └── InMemoryOrderRepository.php # Реализация репозитория в памяти
        │   └── EventBus/           # Механизмы обмена событиями
        │       └── SimpleEventBus.php # Простая реализация шины событий
        └── Api/                     # API слой для взаимодействия с внешним миром
            └── Controllers/         # Контроллеры для обработки HTTP-запросов
                └── OrderController.php # Контроллер заказов (API)
```

## Модули и компоненты (упрощенно)

### 1.1 Domain (Сущности, Value Objects, Бизнес-правила)

- **Order** — агрегат, содержит список OrderItem, управляет статусом и подсчётом суммы.
- **OrderItem** — значение, содержит товар, количество и цену.
- **Client** — значение, содержит сведения о клиенте.
- **Product** — значение, содержит сведения о товаре.
- **Money** — значение, инкапсулирует сумму и валюту.
- **OrderRepositoryInterface** — интерфейс репозитория.

### 1.2 Infrastructure (Реализация репозитория)

- **PDOOrderRepository** — реализует интерфейс, работает с базой данных через PDO.

### 1.3 API (Контроллеры)

- **OrderController** — принимает HTTP-запросы, вызывает бизнес-слой.

### Итог

Этот пример показывает, как в PHP реализовать DDD:

- **Объекты-значения (Value Objects):** Money, Price, OrderStatus
- **Сущности, агрегаты:** Order, OrderItem, Client, Product
- **Репозитории:** интерфейс для хранения заказов
- **Бизнес-сервисы:** OrderService — реализует бизнес-логику
- **Redis:** для кэширования
- **Контроллер API:** — реализует взаимодействия через HTTP
- **PHPUnit:** — для тестирования
